Documentação Técnica – Backend Inicial KINGSTAR
1. Contexto do Projeto

O objetivo do backend é fornecer toda a lógica de um sistema de gestão de pedidos, estoque, reservas e faturamento, integrável via Fastify com banco de dados MySQL/MariaDB. O foco do dia foi criar a estrutura de banco de dados, rotas principais e funções de serviço para pedidos, estoque e relatórios iniciais.

2. Estrutura de Banco de Dados Criada

Tabelas principais:

users → cadastro de usuários do sistema.

customers → cadastro de clientes.

suppliers → cadastro de fornecedores.

categories → categorização de produtos.

products → produtos, vinculados a categorias.

warehouses → armazéns.

locations → localização de produtos dentro do armazém.

inventory → quantidade disponível e reservada de produtos.

inventory_movements → histórico de movimentações de estoque.

orders → pedidos de clientes.

order_items → itens dentro de cada pedido.

stock_reservations → reservas de estoque por pedido.

picking_orders e picking_items → picking de pedidos

 ✅ O que já concluímos

Banco de Dados

Estrutura completa das tabelas principais: users, customers, products, orders, order_items, inventory, stock_reservations, warehouses, locations.

Relacionamentos com FOREIGN KEY e constraints de integridade (evitando problemas de duplicate e FK).

Dados de teste populados, já permitindo criar pedidos, reservar estoque e listar informações.

Service Functions

createOrder(data) → cria pedido, processa itens, reserva estoque e calcula total.

listOrders() → lista pedidos em ordem decrescente.

cancelOrder(orderId) → cancela pedido, libera estoque e atualiza reservas.

Lock de estoque e transações garantem consistência (FOR UPDATE + beginTransaction()).

Testes básicos executados

Criação de pedidos via service.

Cancelamento de pedidos e liberação de estoque.

Conferência de estoque (quantity vs reserved_quantity).

Listagem de pedidos em ordem correta.

Estrutura pronta para Fastify

Rotas podem chamar diretamente essas funções de service.

IDs de teste já definidos e funcionais.
